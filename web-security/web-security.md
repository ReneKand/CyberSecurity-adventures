# Web Security

This directory covers web application security — vulnerabilities, exploitation techniques, real-world incidents, and lab-based experiments.

## Why this directory exists

The web is the most exposed attack surface in modern infrastructure. Understanding how web applications fail — and why — is foundational to both offensive and defensive security work. This directory exists to build and document that understanding through hands-on practice and research into real events.

## What the need is

Web vulnerabilities are well-documented in the abstract, but understanding them in practice requires working through actual examples. Labs build the muscle memory; real-world incidents provide the context for why these issues matter and what the consequences look like at scale.

## What to expect

The directory contains one subdirectory:

- `Hacks/` — lab-based write-ups and experiments with common web vulnerabilities such as SQL injection, XSS, authentication flaws, and input validation issues.

The focus is on the practical side: finding, exploiting, and understanding web vulnerabilities in controlled environments. Real-world incident analysis lives in `concepts/Articles/`.
