# Hacks

This directory contains hands-on experiments and lab-based write-ups covering common web application vulnerabilities.

## Why this directory exists

Reading about SQL injection or XSS does not teach you what it actually feels like to find and exploit one. Working through labs — even simple ones — builds the practical intuition for recognizing vulnerable patterns in real code. This directory documents that practice.

## What the need is

Web vulnerabilities are often described as solved problems, yet they keep appearing in production systems. Understanding why requires more than knowing the definition — it requires seeing how developers introduce them, how easy they are to miss in review, and how straightforward they are to exploit once found. Hands-on practice makes that concrete.

## What to expect

Each file covers a specific vulnerability class or a lab-based scenario. Entries include:

- The vulnerability being explored and why it matters
- The environment or lab used
- Steps taken to identify and exploit the issue
- What made it possible — the code or configuration flaw behind it
- Takeaways on detection, prevention, or related patterns to watch for

Vulnerability classes covered include SQL injection, XSS, authentication flaws, misconfigurations, and input validation issues, among others as they come up in practice.
