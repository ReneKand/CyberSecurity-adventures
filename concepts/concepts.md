# Concepts

This directory contains write-ups on security fundamentals, theory, and the mental models that underpin practical offensive and defensive work.

## Why this directory exists

Tools and techniques change. Concepts do not. Understanding threat modeling, attack surfaces, privilege boundaries, and defense-in-depth means being able to reason about security problems you have never seen before — not just apply known exploits to known vulnerabilities. This directory exists to build that foundation.

## What the need is

It is easy to get caught up in the practical side of security work and skip the theory. But the theory is what allows you to generalize. Knowing why a vulnerability class exists is more useful than memorizing its exploit. Knowing what an attack surface is means you can identify one in any system, not just the ones you have practiced on.

## What to expect

Short write-ups on security fundamentals, including:

- Threat modeling — how to think about what an attacker wants, what they have access to, and what stands between them and their goal
- Attack surfaces — identifying where a system is exposed and what that exposure means
- Defense-in-depth — how layered controls work and why any single control is insufficient
- Privilege boundaries — what separates user from root, unprivileged from privileged, and how those boundaries get crossed
- Linux capabilities and permissions — how the permission model works and where it breaks down

Entries are kept concise. The goal is a clear mental model, not an exhaustive reference.
