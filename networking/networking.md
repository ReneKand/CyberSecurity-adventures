# Networking

This directory covers network scanning, enumeration, infrastructure analysis, and the concepts behind how systems communicate — and how those communications can be investigated or exploited.

## Why this directory exists

Almost every engagement starts with the network. Before you can attack a service, you need to find it. Before you can exploit a misconfiguration, you need to understand the protocol. Networking knowledge is not optional in security — it is the foundation that everything else is built on.

## What the need is

It is easy to run nmap and move on. The harder part is understanding what the output means: why a port is open, what service is likely running, what that service's known weaknesses are, and how to pivot from there. This directory exists to build and document that deeper understanding rather than treating enumeration as a checkbox.

## What to expect

Notes and write-ups on topics including:

- Network scanning and enumeration methodology
- Service and port analysis — what is running and what it means
- Protocol behavior and how it can be abused
- Privilege escalation through network services or misconfigurations
- Misconfigured Linux and Windows systems in lab environments

Entries are drawn from lab work, HTB machines, and concepts encountered during research. The focus is on understanding the network layer well enough to make informed decisions during an engagement.
