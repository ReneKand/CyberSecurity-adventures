# Challenges

This directory contains write-ups for standalone HTB challenges — individual tasks that focus on a specific skill or vulnerability class rather than a full machine compromise.

## Why this directory exists

Challenges isolate specific techniques in a way that full machines do not. A forensics challenge forces focus on packet analysis or file carving. A crypto challenge demands understanding of the underlying algorithm. They are useful for targeted skill development outside of the end-to-end machine flow.

## What the need is

Challenges are often where unfamiliar territory gets explored. Documenting them means I have a reference for techniques I would otherwise forget, especially in categories I visit less frequently like reverse engineering or steganography.

## What to expect

Each file covers one challenge and is named accordingly. Entries typically include:

- Challenge category and brief description
- Initial observations and approach
- Tools used and why
- Key insight that led to the solution
- Takeaway or concept worth noting for future reference

Like the machine write-ups, flags and sensitive details are redacted. The write-up exists to capture thinking, not answers.
